### 1️⃣ 비즈니스 & 전략

#### 1.1 프로젝트 배경 및 비전

**Q1. 이 프로젝트를 시작하게 된 배경은 무엇인가요?**
```
답변:
사내 직원들이 각자 다른 방식(메모장, 엑셀, 종이 수첩 등)으로 할일을 관리하다 보니
체계적인 업무 관리가 어렵고, 중요한 할일을 놓치는 경우가 자주 발생합니다.
특히 국경일이나 회사 휴무일 등의 공통 일정을 개인이 따로 챙겨야 해서 불편합니다.
통합된 할일 관리 도구가 필요하다는 의견이 지속적으로 제기되어 프로젝트를 시작하게 되었습니다.
```

**Q2. 현재 사내에서 할일 관리는 어떻게 하고 있나요?**
```
답변:
- 개발팀: Notion, Jira 티켓, 개인 메모장 혼용
- 기획팀: Excel 스프레드시트, 종이 수첩
- 영업팀: 이메일 플래그, 스마트폰 메모 앱
- 관리팀: 종이 다이어리, 화이트보드

→ 팀마다, 개인마다 다른 도구를 사용하고 있으며, 공통 일정(국경일, 회사 행사)은
  별도로 메신저나 이메일로 공지받아 각자 수동으로 관리합니다.
```

**Q3. 현재 방식의 가장 큰 문제점은 무엇인가요?**
```
답변:
1. 일관성 부족: 팀/개인마다 다른 도구를 써서 업무 현황 파악이 어려움
2. 할일 누락: 여러 도구에 분산되어 중요한 할일을 놓치는 경우 발생
3. 공통 일정 관리 불편: 국경일, 회사 행사 등을 개인이 일일이 입력해야 함
4. 복구 불가: 실수로 삭제한 할일을 되돌릴 방법이 없음
5. 접근성 문제: 외부 도구 사용 시 회사 밖에서 접근이 제한되거나 보안 우려
```

**Q4. 이 제품이 완성되었을 때 가장 기대되는 변화는 무엇인가요?**
```
답변:
- 전 직원이 하나의 통합된 플랫폼에서 할일을 관리하여 업무 효율성 향상
- 국경일/회사 행사 등 공통 일정이 자동으로 표시되어 개인이 따로 챙길 필요 없음
- 실수로 삭제한 할일도 휴지통에서 복구 가능하여 안심하고 사용
- 개인별 할일 완료율, 업무 패턴 파악으로 생산성 개선
- 사내 전용 도구로 보안 걱정 없이 안전하게 업무 정보 관리
```

기타:

### 7️⃣ 기타 요구사항

**Q40. 특별히 강조하고 싶은 요구사항이나 제약사항이 있나요?**
```
답변:

1. **속도가 생명**:
   할일 관리 도구는 빠르게 추가하고 빠르게 체크하는 게 핵심입니다.
   로딩이나 불필요한 클릭이 많으면 사용하지 않게 됩니다.

2. **단순함 유지**:
   기능이 많아지면 복잡해져서 오히려 사용성이 떨어집니다.
   MVP에서 정말 필요한 기능만 넣고, 나머지는 사용자 피드백 보고 추가하겠습니다.

3. **모바일 필수**:
   회의 중, 이동 중에도 할일을 추가할 수 있어야 합니다.
   반응형 웹으로 모바일 대응은 MVP부터 필수입니다.

4. **보안 중요**:
   내부 도구지만 업무 정보가 담기므로 기본적인 보안은 반드시 지켜야 합니다.
   HTTPS, 비밀번호 암호화는 필수입니다.

5. **국경일 자동 표시**:
   이게 다른 도구와의 차별점입니다. 개인이 일일이 입력하지 않아도
   국경일, 회사 휴무일이 자동으로 표시되어야 합니다.
```

**Q41. 반드시 피해야 할 것이 있나요?** (예: 복잡한 UI, 강제 튜토리얼 등)
```
답변:

피해야 할 것들:
1. **복잡한 UI**: Notion처럼 기능이 많고 복잡하면 안 됩니다. 처음 보는 사람도 3분 안에 사용 가능해야 합니다.

2. **느린 로딩**: 페이지 전환마다 로딩 스피너 돌아가면 답답합니다. SPA로 빠르게 반응해야 합니다.

3. **강제 온보딩**: 긴 튜토리얼이나 강제 가이드는 사용자를 짜증나게 합니다. 건너뛰기 가능한 간단한 안내만.

4. **과도한 알림**: 알림이 너무 많으면 오히려 무시하게 됩니다. 정말 필요한 알림만 (마감일 D-1).

5. **필수 입력 항목 많음**: 할일 추가할 때 제목만 입력해도 저장 가능해야 합니다. 나머지는 선택.

6. **데스크톱 전용**: PC에서만 쓸 수 있으면 회의 중, 외출 중 사용 못 합니다. 모바일 필수.

7. **외부 의존성**: 외부 API나 서비스에 의존하면 장애 시 같이 멈춥니다. 자체 완결성 유지.

8. **너무 많은 기능**: 사용하지 않는 기능이 많으면 오히려 불편합니다. 80%가 사용하는 기능만 제공.
```

**Q42. 참고할 만한 레퍼런스나 벤치마크가 있나요?**
```
답변:

UI/UX 레퍼런스:
1. **Todoist** (https://todoist.com)
   - 벤치마크 이유: 간결한 UI, 빠른 할일 추가, 자연어 입력
   - 참고 포인트: 메인 화면 레이아웃, 우선순위 색상, 체크 애니메이션

   - 벤치마크 이유: 심플하고 아름다운 디자인, 직관적인 흐름
   - 참고 포인트: 할일 상세 화면, 계층 구조, 애니메이션

4. **Google Keep** (https://keep.google.com)
   - 벤치마크 이유: 간단한 메모, 색상 구분, 모바일 최적화
   - 참고 포인트: 카드 레이아웃, 색상 태그, 검색

기능 레퍼런스:
- 휴지통: Gmail의 휴지통 (30일 자동 삭제)
- 완료 체크: Todoist의 체크 애니메이션
- 공통 일정: Google Calendar의 국경일 표시

피해야 할 레퍼런스:
- Notion: 너무 복잡, 기능 과다
- Asana/Jira: 프로젝트 관리 중심, 개인용에 부적합
- Microsoft To Do: UI가 구식, 느림
```
